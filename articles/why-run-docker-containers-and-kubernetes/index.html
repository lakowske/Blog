Why run Docker containers using Kubernetes?

Posted by: Seth Lakowske

Published: 2016-04-25

  • Service isolation without the heavyweight VM. Adding a service to a server does not affect other services on the server
  • Lends itself to immutable infrastructure, so services are reloadable without missing configuration when a server goes down
  • Immutable containers can be created which improve security. Data can only be mutated in programmer specified volumes, so root kits can't be installed even if the system is penetrated.
  • Services can be run on a large cluster of nodes making it more reliable
  • Code can be broken out into smaller micro services that are easier to learn/release/update
  • Some environments will only host in private clusters and not public clouds
  • Both Docker and Kubernetes are free to use on a workstation or public/private cloud.
  • Access to Docker hub leverages the work of the opensource community
  • Developers can run containers locally, rebuilding verifying after each commit on a system that mirrors production
